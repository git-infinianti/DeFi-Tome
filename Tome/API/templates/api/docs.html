üîê Authentication

All endpoints marked as "requires authentication" require an API key. You can create and manage your API keys in the Settings page.

How to Authenticate:

Include your API key in one of the following headers: 

# Option 1: Authorization header with Bearer token
Authorization: Bearer YOUR_API_KEY_HERE

# Option 2: X-API-Key header
X-API-Key: YOUR_API_KEY_HERE

Example with curl: 

curl -X POST https://your-domain.com/api/v1/contracts/ \
  -H "Authorization: Bearer YOUR_API_KEY_HERE" \
  -H "Content-Type: application/json" \
  -d '{"name": "MyContract", "contract_address": "CONTRACT_ADDR"}'

üìú Solidity Smart Contracts

GET /api/v1/contracts/

List all deployed Solidity contracts (stateless contracts on Evrmore).

Query Parameters:

page integer Page number for pagination (default: 1)
per_page integer Results per page (default: 20)
deployer integer Filter by deployer user ID
POST /api/v1/contracts/ üîí Auth Required

Deploy a new Solidity contract. Requires API key authentication.

Request Body:

name string Contract name (required)
contract_address string Evrmore asset name or identifier (required)
source_code string Solidity source code (optional)
abi array Contract ABI JSON (optional)
description string Contract description (optional)
GET /api/v1/contracts/{contract_id}/

Get detailed information about a specific contract including source code, ABI, and recent interactions.

POST /api/v1/contracts/{contract_id}/interact/ üîí Auth Required

Interact with a deployed contract by calling a function. Requires API key authentication.

Request Body:

function_name string Name of function to call (required)
parameters object Function parameters as key-value pairs

ü™ô Evrmore Assets

GET /api/v1/assets/

List all assets on the Evrmore blockchain.

Query Parameters:

asset string Asset name filter (default: "*" for all)
verbose boolean Include detailed info (default: false)
count integer Results per page (default: 50)
GET /api/v1/assets/{asset_name}/

Get detailed information about a specific asset.

POST /api/v1/assets/issue/ üîí Auth Required

Issue a new asset on the Evrmore blockchain. Requires API key authentication.

Request Body:

asset_name string Name of the asset (required)
qty number Quantity to issue (required)
units integer Divisibility 0-8 (default: 0)
reissuable boolean Can be reissued (default: true)
ipfs_hash string IPFS hash for metadata (optional)
POST /api/v1/assets/transfer/ üîí Auth Required

Transfer an asset to another address. Requires API key authentication.

Request Body:

asset_name string Name of the asset (required)
qty number Quantity to transfer (required)
to_address string Recipient address (required)
message string Optional message (optional)

üîó Blockchain Operations

GET /api/v1/blockchain/info/

Get general blockchain information including chain type, block count, and difficulty.

GET /api/v1/blockchain/block/{block_hash}/

Get information about a specific block by its hash.

Query Parameters:

verbosity integer Level of detail (default: 1)
GET /api/v1/blockchain/address/{address}/balance/

Retrieve the current balance of a blockchain address. 

{
  "success": true,
  "address": "EVRTestAddress123456789",
  "balance": {
    "balance": 1000000000,
    "received": 1000000000
  }
}
GET /api/v1/blockchain/address/{address}/transactions/

Get transaction IDs for a specific address.

Query Parameters:

start integer Start block height (optional)
end integer End block height (optional)
GET /api/v1/blockchain/address/{address}/utxos/

Get unspent transaction outputs (UTXOs) for an address.

üí¨ Messaging

POST /api/v1/messages/send/ üîí Auth Required

Send a message to a channel on the Evrmore blockchain. Requires API key authentication.

Request Body:

channel_name string Name of the channel (required)
ipfs_hash string IPFS hash of the message (required)

GET /api/v1/messages/

View all messages from subscribed channels.

GET /api/v1/messages/channels/
